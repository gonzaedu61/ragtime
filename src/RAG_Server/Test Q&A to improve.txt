(ragtime) PS C:\dev\ragtime> rag "What are the core steps of Production Planning?" pepe.json                 
Here are the core steps of production planning as described in the context:

• Determine the planning sequence  
  – Based on the product-part numbering from the technical description  
  – And the calculation sort order from the operation master

• Assign the planning stage  
  – For example, stage 1 = sheet parts (products 1–89), stage 2 = finishing (products 90–99)

• Calculate the earliest start date and time  
  – Of the first production operation (e.g. “Drucken”/printing)

• Automatically schedule the products in that sequence  
  – The system generates or copies planning positions into the plan

• Derive the scheduled production end (“ProdEnde”)  
  – Date and time when the order is expected to finish

• Apply standard deadline keys  
  – “Kunde kommt” (customer arrival) target date/time  
  – “Vorgabe Fertigung” (production target) date/time

• Check material availability status  
  – According to the “Vorgabe Material” schedule key of the planning type

The Almendro ERP derives its shop‐floor sequence from two settings:

1. Product-part numbering  
   – The system reads the “Product-Teil-Nummer” you’ve assigned in the Technical Description and sorts all parts in ascending order of that number.

2. Operation sort key (Kalkulationssort) in the operation master
   – Within each part, Almendro then orders the individual operations by the “Kalkulationssort” field.

In practice you’ll often simply number your cost centres so that their ascending cost-centre numbers also reflect the desired sequence of operations. If your cost-centre numbers already run in the correct order, no further configuration is required.

(ragtime) PS C:\dev\ragtime> rag "What is the Terminschlüssel Stammdaten? Answer in English please" pepe.json
The “Terminschlüssel” in the master data is simply the scheduling‐key record you must maintain under the Master Data table called “Terminschlüssel” at the “Order” (Auftrag) level. In practice:

• It defines your planning key for each order
• It drives the running number display in the planning mask
• It is also the key set by the “Change EDate” function when you adjust an order’s end date

So whenever you talk about Terminschlüssel Stammdaten, you’re referring to that scheduling‐key entry in the Master Data (“Terminschlüssel” table, level = “Order”).
