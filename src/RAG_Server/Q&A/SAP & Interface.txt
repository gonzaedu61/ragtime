(ragtime) PS C:\dev\ragtime> rag "Give me some conceptual description of the ASCII-file interface" pepe.json
Here’s a high-level, conceptual overview of how the ASCII-file interface works:

• File‐level ordering  
  – Records (bookings) are processed strictly in the order they appear in the ASCII file.  
  – Only data for one cost-centre is contained in each file.  

• Record structure  
  – Each booking is one “record” (one line) in the file.  
  – The first four fields are always mandatory and always in the same order:  
     1. Date (tt.mm.yyyy)  
     2. Time (hh:mm:ss)  
     3. Cost centre  
     4. Booking type
  – Any further fields depend on the booking type. Which fields are “must” versus “can” is defined in the ASCII‐interface specification.

• Field and line delimiters
  – Fields are separated by a TAB character (ASCII 9).
  – Lines end with LF or CR+LF.

• Error/older‐data handling
  – If older data (e.g. an earlier ASCII file) arrives out of sequence, it will not be re-processed or booked once newer data for that cost centre has already been taken in.
